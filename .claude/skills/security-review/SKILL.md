# Security Review Skill

version: 1.0

description: 对当前分支或指定代码执行安全审查，覆盖 OWASP Top 10，输出按严重程度排列的结构化报告。

## 触发条件

用户明确要求安全审查，或代码涉及以下任一场景时触发：
- 认证 / 授权 / 会话管理
- 用户输入处理（表单、URL 参数、文件上传）
- 数据库查询 / ORM 操作
- 加密 / 密钥 / 证书管理
- 外部 HTTP 请求 / 第三方 API 调用
- 序列化 / 反序列化

## 执行步骤

1. **注入类漏洞**：检查 SQL 注入、命令注入、LDAP 注入，确认所有查询使用参数化或 ORM
2. **XSS 防护**：验证所有用户输入在输出到 HTML/JS 前已转义，CSP 头是否配置
3. **失效的身份认证**：检查密码存储（bcrypt/argon2）、token 有效期、会话固定攻击
4. **敏感数据泄露**：确认日志、错误响应、注释中不包含密钥、PII、内部路径
5. **权限与访问控制**：确认最小权限原则，检查水平/垂直越权风险
6. **安全配置错误**：检查默认凭证、调试模式、不必要的端口/服务暴露
7. **SSRF**：检查服务端发起的外部请求是否对目标 URL 做了白名单校验
8. **不安全反序列化**：检查 JSON/XML/二进制反序列化入口，确认输入来源可信
9. **已知漏洞组件**：列出依赖中存在已知 CVE 的版本（结合 `go list -m all` 或 `npm audit`）
10. **OWASP Top 10 对照**：输出完整检查表，标注每项状态（通过 / 警告 / 高危 / 不适用）

## 输出格式

按 CVSS 3.1 评分从高到低排列，每个问题包含：

```
[严重程度] CVSS x.x — 问题标题
位置：文件名:行号
描述：问题说明
风险：可能造成的危害
修复建议：具体修复方案
```

严重程度分级：
- 🔴 高危（CVSS ≥ 7.0）：必须在合入前修复
- 🟡 中危（CVSS 4.0–6.9）：本迭代内修复
- 🔵 低危（CVSS < 4.0）：记录并跟踪
