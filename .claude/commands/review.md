# Code Review

请对当前 git 工作区的改动执行完整的代码审查。

**获取改动内容：**
先运行 `git diff HEAD` 获取未提交的改动；若工作区为干净状态，则运行 `git diff HEAD~1 HEAD` 获取最近一次提交的改动。

**审查维度：**
1. 安全漏洞（SQL 注入、命令注入、权限绕过）
2. 错误处理完整性（Go 代码中不得用 `_` 忽略错误，使用 `%w` 包装）
3. 测试覆盖率（核心逻辑是否有对应测试，遵循 `.claude/rules/test.md`）
4. 代码风格规范（遵循 `.claude/rules/code-style.md`）
5. 性能影响（goroutine 泄露、不必要的内存分配、N+1 查询）

用中文输出结构化审查报告，按严重程度排列问题。
